# Basic GitHub Operations

## Introduction to GitHub at Cortland Credit Group Inc.

Welcome to the GitHub guide tailored for your role at Cortland Credit Group Inc. This manual will guide you through basic GitHub operations, focusing on accessing and contributing to our private repositories.

## Setting Up Your GitHub Account for Cortland Credit Group Inc.

1. **Create a GitHub Account:** If you haven't already, sign up at [GitHub](https://github.com/).

2. **Install Git:** Download and install Git from [git-scm.com](https://git-scm.com/).

3. Configure Git:

    Open your terminal (Command Prompt or Git Bash) and set up your user name and email:

   ```bash
   git config --global user.name "Your Name"
   git config --global user.email "your_email@example.com"
   ```

## Gaining Access to Private Repositories

- **Request Access:** You will need specific permissions to access private repositories of Cortland Credit Group Inc. Please request access from your team lead or IT administrator.
- **Accept Invitation:** Once you are granted access, you'll receive an invitation via email. Accept this invitation to access the repositories.

## Cloning a Repository Using GitHub Desktop

- **Clone a Repository:** In GitHub Desktop, go to `File` > `Clone repository`, select the `cortland-credit-group-inc` organization, and choose the repository you wish to clone.

## Managing Branches with GitHub Desktop

- **Creating a Branch:** Click on the `Current branch` dropdown at the top, then `New Branch`. Name your branch and create it.
- **Switching Branches:** Use the `Current branch` dropdown to switch between branches.

## Making Changes and Committing

- **Making Changes:** Edit files in your preferred editor. GitHub Desktop will track these changes.
- **Committing Changes:** In GitHub Desktop, write a commit message describing your changes and click `Commit to [branch-name]`.

## Pushing and Pulling Changes with GitHub Desktop

- **Pushing Changes:** Click `Repository` and then `Push` to upload your changes to GitHub.
- **Pulling Changes:** Click `Repository` and then `Pull` to update your local repository with the latest changes from GitHub.

## Merging Branches Using GitHub Desktop

- **Merging Changes:** Switch to the branch you want to merge into (usually `main`). Click `Branch` in the menu, then `Merge into current branch`. Select the branch you want to merge from.

## Handling Merge Conflicts

- **Resolving Conflicts:** If there are conflicts, GitHub Desktop will notify you. Resolve these conflicts in your file editor, then commit the changes.

## Additional Resources

- **GitHub Desktop Documentation:** Explore detailed guides and troubleshooting tips at [GitHub Desktop Docs](https://docs.github.com/en/desktop).
